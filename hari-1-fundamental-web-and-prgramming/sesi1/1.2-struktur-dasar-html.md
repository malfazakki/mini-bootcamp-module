**HTML** adalah singkatan dari **HyperText Markup Language**. Ini adalah bahasa standar untuk membuat halaman web.

**ğŸ’¡ Analogi Sederhana:**
Jika web adalah rumah:

## ğŸ—ï¸ Struktur Dasar Dokumen HTML5

Setiap halaman web dimulai dengan struktur dasar seperti ini:

```html
<!DOCTYPE html>
<html lang="id">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Halaman Pertama Saya</title>
</head>
<body>
    <h1>Halo Dunia!</h1>
    <p>Ini adalah paragraf pertama saya.</p>
</body>
</html>
```

### ğŸ” Penjelasan Baris per Baris:

| **Baris Kode** | **Penjelasan** |
|----------------|----------------|
| `<!DOCTYPE html>` | Memberitahu browser bahwa ini adalah dokumen HTML5 (versi terbaru) |
| `<html lang="id">` | Tag pembuka HTML. `lang="id"` artinya bahasa Indonesia |
| `<head>` | Bagian kepala (tidak terlihat di halaman). Berisi informasi metadata |
| `<meta charset="UTF-8">` | Mengatur karakter encoding agar bisa menampilkan huruf Indonesia dengan benar |
| `<meta name="viewport"...>` | Agar tampilan bagus di semua ukuran layar (HP, tablet, desktop) |
| `<title>` | Judul yang muncul di tab browser |
| `</head>` | Penutup bagian head |
| `<body>` | Bagian tubuh (konten yang terlihat di halaman) |
| `<h1>` | Heading level 1 (judul paling besar) |
| `<p>` | Paragraf |
| `</body>` | Penutup body |
| `</html>` | Penutup HTML |

## ğŸ“¦ Elemen-Elemen Penting HTML (Yang Wajib Dikuasai)

### 1ï¸âƒ£ **Heading - Judul**

HTML punya 6 level heading, dari yang terbesar (h1) sampai terkecil (h6):

```html
<h1>Judul Utama (Paling Besar)</h1>
<h2>Sub Judul</h2>
<h3>Sub-Sub Judul</h3>
<h4>Judul Level 4</h4>
<h5>Judul Level 5</h5>
<h6>Judul Terkecil</h6>
```

**ğŸ“Œ Best Practice:**
- Gunakan **hanya satu `<h1>`** per halaman (untuk judul utama)
- Gunakan h2, h3, dst. untuk sub-judul secara berurutan

---

### 2ï¸âƒ£ **Paragraf**

```html
<p>Ini adalah paragraf pertama.</p>
<p>Ini adalah paragraf kedua.</p>
```

Jika Anda ingin teks pindah baris di dalam 1 paragraf:
```html
<p>Baris pertama<br>Baris kedua</p>
```

---

### 3ï¸âƒ£ **Link (Anchor)**

Link untuk pindah ke halaman lain:

```html
<!-- Link ke website lain -->
<a href="https://www.google.com">Klik untuk ke Google</a>

<!-- Link ke halaman lain di website kita -->
<a href="tentang.html">Tentang Kami</a>

<!-- Link yang membuka di tab baru -->
<a href="https://youtube.com" target="_blank">Buka YouTube</a>
```

**Penjelasan Atribut:**
- `href` = Alamat tujuan (hyper-reference)
- `target="_blank"` = Buka di tab baru

---

### 4ï¸âƒ£ **Gambar**

```html
<img src="foto-saya.jpg" alt="Foto profil saya" width="300">
```

**Penjelasan:**
- `src` = Sumber gambar (source). Bisa berupa file lokal atau URL
- `alt` = Deskripsi gambar (muncul jika gambar gagal dimuat)
- `width` = Lebar gambar dalam pixel

**Contoh dengan gambar dari internet:**
```html
<img src="https://via.placeholder.com/300" alt="Placeholder">
```

---

### 5ï¸âƒ£ **List (Daftar)**

#### **Unordered List (Bullet Points):**
```html
<ul>
    <li>Apel</li>
    <li>Jeruk</li>
    <li>Mangga</li>
</ul>
```

**Hasil:**
- Apel
- Jeruk
- Mangga

#### **Ordered List (Numbering):**
```html
<ol>
    <li>Bangun tidur</li>
    <li>Mandi</li>
    <li>Sarapan</li>
</ol>
```

**Hasil:**
1. Bangun tidur
2. Mandi
3. Sarapan

---

### 6ï¸âƒ£ **Container: Div & Span**

**`<div>`** = Container untuk mengelompokkan elemen (block-level, punya baris sendiri)

```html
<div>
    <h2>Tentang Saya</h2>
    <p>Nama saya Budi.</p>
</div>
```

**`<span>`** = Container kecil untuk styling teks inline (tidak membuat baris baru)

```html
<p>Saya suka warna <span style="color: blue;">biru</span>.</p>
```

---

### 7ï¸âƒ£ **Semantic HTML (HTML Bermakna)**

Menggunakan tag yang lebih deskriptif daripada `<div>` biasa:

```html
<header>
    <h1>Website Saya</h1>
    <nav>
        <a href="#home">Home</a>
        <a href="#about">About</a>
    </nav>
</header>

<main>
    <article>
        <h2>Artikel Pertama</h2>
        <p>Isi artikel...</p>
    </article>
</main>

<footer>
    <p>&copy; 2026 Website Saya</p>
</footer>
```

**Keuntungan Semantic HTML:**
- Lebih mudah dibaca oleh developer lain
- Lebih baik untuk SEO (Google bisa lebih paham struktur web Anda)
- Lebih accessible (ramah untuk screen reader)
