## ğŸ“„ Deployment dengan GitHub Pages

**Alternative: GitHub Pages** (jika tidak mau pakai Vercel).

### **Step 1: Pastikan Repo Public**

Di GitHub, repo harus **public** untuk free tier.

### **Step 2: Enable GitHub Pages**

1. Buka repository di GitHub
2. Click **Settings**
3. Scroll ke **Pages** (sidebar kiri)
4. Source: **Deploy from a branch**
5. Branch: **main** (atau master)
6. Folder: **/ (root)**
7. Click **Save**

### **Step 3: Tunggu Deploy**

GitHub akan build & deploy (~1-2 menit).

URL:
```
https://USERNAME.github.io/REPOSITORY-NAME/
```

**Contoh:**
```
https://johndoe.github.io/my-portfolio/
```

## ğŸ”„ Environment Variables di Frontend

**Masalah:**
```javascript
// âŒ Hardcoded API URL
const API_URL = 'http://localhost:3000/api';
```

**Di production, API URL berbeda!**

**Solusi: Environment Variables**

### **Vercel Environment Variables:**

1. **Project Settings â†’ Environment Variables**
2. Tambahkan:
   ```
   Key: VITE_API_URL (atau API_URL)
   Value: https://mybackend.onrender.com/api
   ```
3. Save & Re-deploy

### **Di Code:**

```javascript
// Baca dari environment variable
const API_URL = import.meta.env.VITE_API_URL || 'http://localhost:3000/api';

// Atau pakai conditional
const API_URL = window.location.hostname === 'localhost'
    ? 'http://localhost:3000/api'
    : 'https://mybackend.onrender.com/api';
```

## ğŸ› Troubleshooting Deployment

### **1. Build Error**

**Error:** `Build failed`

**Solusi:**
- Cek console log di Vercel dashboard
- Pastikan tidak ada typo di HTML/CSS/JS
- Test di local dulu sebelum deploy

---

### **2. Blank Page / White Screen**

**Error:** Website ngebuka tapi kosong

**Solusi:**
- Buka DevTools â†’ Console
- Cek error messages
- Pastikan path file benar (case-sensitive!)
- Jangan pakai absolute path (`C:\Users\...`)

---

### **3. API Request Gagal (CORS)**

**Error:** `CORS policy blocked`

**Solusi:**
- Pastikan backend sudah enable CORS
- Update API URL di frontend ke production URL
- Cek Network tab untuk lihat request

---

### **4. 404 Not Found**

**Error:** Routing tidak bekerja

**Solusi untuk SPA (Single Page App):**
- Tambahkan `vercel.json`:
```json
{
  "rewrites": [
    { "source": "/(.*)", "destination": "/" }
  ]
}
```

## ğŸ¯ Best Practices Deployment

### **1. Test Sebelum Deploy**
```bash
# Local testing
npm run dev
# atau buka dengan Live Server
```

### **2. Gunakan Git Branching**
```bash
# Develop di branch terpisah
git checkout -b feature/new-feature

# Merge ke main setelah tested
git checkout main
git merge feature/new-feature
git push
```

### **3. Performance Optimization**
- Minify CSS & JavaScript
- Optimize images (compress, use WebP)
- Lazy load images
- Use CDN untuk libraries

### **4. Security**
- HTTPS otomatis enabled (Vercel/GitHub Pages)
- Jangan expose API keys di frontend!
- Use environment variables
